This is a body paragraph in `text-l` (18px/165%). Articles use a narrow measure for comfortable reading. **Bold text** for emphasis, *italic* for asides or titles. Both can combine: ***bold italic***. Inline `code` renders in monospace.

A second paragraph to show vertical rhythm between blocks. Links look [like this](#) within the flow of text. The article body supports all standard markdown plus the components listed below.

## Heading 2—Section Title

Use `##` for major sections. This is the primary structural heading inside the article body. The article title (h1) comes from frontmatter via ArticleHeader.

### Heading 3—Subsection

Use `###` for subsections within a section. Don't skip levels.

#### Heading 4—Minor Heading

Use `####` sparingly. If you need this many levels, the article might need restructuring.

**Frontmatter fields:** `title` (required), `subtitle` (optional), `date` (required), `cover` (optional—hero image above body), `draft` (boolean), `theme` (`"dark"` or `"light"`, defaults to light).

**Components:** `Figure` (src, alt, caption?), `Divider` (label?, spaceAbove?, spaceBelow?, variant?).

**Markdown:** Headings (h2–h4), paragraphs, bold, italic, links, unordered/ordered lists, blockquotes, fenced code blocks, inline code, horizontal rules.

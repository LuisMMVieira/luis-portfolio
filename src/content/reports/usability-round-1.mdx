<ReportSection number="04" title="Key Quotes" description="Direct participant feedback that shaped the issue prioritization.">

<div class="report-quote-grid">
<Callout title="P01 on the search bar:">

"I don't understand the difference between 'Search and transfer a domain' to the search and transfer button. When I click to start typing it shows more options that I was expecting, it gives options that I didn't know that existed — this can be confusing."

</Callout>
<Callout title="P02 on bulk transfer error handling:">

"I copy/pasted the three domains in the input field because I thought the 'transfer' option would be efficient enough to identify them — like how email fields handle multiple addresses. I was expecting the same happened here."

</Callout>
<Callout title="P02 on transfer confidence:">

"I never done a transfer before. I don't know if the process will put my website offline, how much time and steps it will take. I wish it had here some bullet points with that information to help me feel more confident."

</Callout>
<Callout title="P03 on workflow consistency:">

"The transfer process is so fast that I'm afraid I'm doing something wrong. I would like to have some information to reassure that everything is ok — that I won't be charged twice, for example."

</Callout>
</div>

<ReportSection number="05" title="Design Implications" description="What the data points to — actionable next steps.">

<Annotation number="1" title="The search bar is overloaded." text="One input doing search + transfer + bulk transfer confuses everyone. Users need clearly separated flows — search is about discovery, transfer is about moving something they already own. These are different mental models forced into one UI element." />

<Annotation number="2" title="Error handling needs to be immediate and inline." text="Silent failures and delayed error messages create distrust. Domains with format errors should be caught on input, not after submission. Auto-stripping 'https://' is table stakes." />

<Annotation number="3" title="Transfer discoverability is a real gap." text="Users can't find the transfer entry point without scrolling or guessing. A dedicated menu item or prominent CTA on the domains page would reduce friction before the transfer flow even starts." />

<Annotation number="4" title="Trust is a conversion lever." text="Multiple participants expressed anxiety about what happens during a transfer — downtime, double billing, timelines. Reassurance copy and process transparency are low-effort, high-impact fixes." />
